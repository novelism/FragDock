## Data Preparation

The building block dataset is **not distributed** with this repository due to licensing restrictions.  
Users must prepare their own `building_blocks.csv` file before running FragDock.

A template file is provided to show the required format:

```text
building_blocks_template.csv
```

The template contains the required column names:

```csv
Mol_ID,SMILES
```

Each row in `building_blocks.csv` should contain a unique molecule identifier and the corresponding SMILES string.

Example:

```csv
Mol_ID,SMILES
BB_000001,CCN
BB_000002,c1ccccc1Br
```

## Preparing `building_blocks.csv`

1. **Obtain building blocks**

   Building blocks can be obtained from commercial or public sources, depending on the user's license and intended use.

   For example, users may obtain building block data from Enamine or ZINC building block catalogs.

   Please ensure that you have the appropriate rights or license to use the selected building block data.

2. **Create `building_blocks.csv`**

   Prepare a CSV file named:

   ```text
   building_blocks.csv
   ```

   The file must contain the following columns:

   ```csv
   Mol_ID,SMILES
   ```

3. **Place the file in the data directory**

   Move the prepared file to:

   ```text
   data/building_blocks.csv
   ```

4. **Run the data preparation script**

   From the repository root directory, run:

   ```bash
   python data/preparation.py
   ```

The processed building block data will be generated by the preparation script and used by FragDock.
